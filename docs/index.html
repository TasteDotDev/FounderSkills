Usage

Just describe your problem. Founder picks the right frameworks and applies them.

Describe your problem

$ founder "We're a 50-person B2B SaaS at $5M ARR. Growth slowed from 3x to 1.5x."

Founder reads your problem, selects 2-4 relevant frameworks (e.g. Four Fits, Growth Accounting, Unit Economics), loads the expert instructions for each, applies them with full rigor, then synthesizes into actionable recommendations — complete with an assumptions table and a pre-mortem.

Target a specific category

$ founder -c strategy "entering the EU market from the US"
$ founder -c finance "should we raise a Series A or bootstrap?"
$ founder -c marketing "how to grow on TikTok as a B2B company"

Use a specific framework

$ founder -f blitzscaling "we're a marketplace with 10k users"
$ founder -f sequoia-pitch-deck "prepare our Series A deck"
$ founder -f cold-email-framework "outbound to enterprise CTOs"

Browse what's available

$ founder list                 # all 15 categories with framework counts
$ founder list strategy        # all 48 strategy frameworks
$ founder list finance          # all 46 finance frameworks

All Commands

founder "your problem"              # auto-route to best frameworks

Growth stall diagnosis

$ founder "We're a 50-person B2B SaaS doing $5M ARR. Growth slowed from 3x to 1.5x."

Let me dig into this. A growth stall at $5M ARR usually means one of four
things: market saturation, PMF decay, GTM inefficiency, or org scaling
friction. Let me pull up the right frameworks to figure out which one.

# Founder calls applyFramework("strategy", "four-fits-framework")
# Founder calls applyFramework("finance", "growth-accounting")
# Founder calls applyFramework("finance", "unit-economics")

## Four Fits Analysis
Your growth stall pattern suggests a Channel-Model fit problem...

## Growth Accounting
Breaking down your growth: new MRR ~$80K, expansion ~$20K, churn ~$45K...

## Assumptions
| # | Assumption        | Confidence | If wrong...              | How to validate          |
|---|-------------------|------------|--------------------------|--------------------------|
| 1 | Monthly churn ~3% | Medium     | Changes entire diagnosis | Pull actual churn data   |
| 2 | CAC ~$500         | Low        | Unit economics may work  | Check last 90 days spend |

## What to do this week
1. Interview 10 recently churned customers (test if churn is product or price)
2. Calculate actual CAC payback from last quarter's data
3. Only then decide whether this is a retention or acquisition problem

Shell Completion

Tab completion for commands, categories, and framework names.

# Bash — add to ~/.bashrc
eval "$(founder completion bash)"

# Zsh — add to ~/.zshrc
eval "$(founder completion zsh)"

# Fish — save to ~/.config/fish/completions/founder.fish
founder completion fish > ~/.config/fish/completions/founder.fish

Then press Tab to autocomplete:

$ founder list str<TAB>  →  founder list strategy
$ founder -f port<TAB>  →  founder -f porters-five-forces
$ founder config <TAB>  →  show  set  setup

How It Works

1. You describe your problem

In plain language. No special syntax. Just tell it what you're dealing with.

2. Founder picks frameworks

The AI reads your problem, decides which 2-4 frameworks fit, and loads the expert instructions for each.

3. Deep analysis, not platitudes

Each framework is applied with full rigor — every section, specific to your situation, with uncertainty flagged.

4. Assumptions surfaced

Every analysis ends with an assumptions table: what was assumed, confidence level, what breaks if wrong, how to validate.

5. Adversarial self-check

Devil's advocate, stress-tested assumptions, and a pre-mortem imagining how the strategy could fail.

6. Actionable next steps

Concrete experiments to run this week, with specific pass/fail criteria before committing resources.

Configuration

Config is stored at ~/.config/founder/config.json. Env vars take priority.

Environment variables (auto-detected)

# Set any of these and founder auto-detects the provider:
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export GOOGLE_GENERATIVE_AI_API_KEY=AIza...
export OPENROUTER_API_KEY=sk-or-...

Config commands

founder config                      # show current config
founder config set provider anthropic # switch provider
founder config set model gpt-5.2     # switch model
founder config set apiKey sk-...      # set API key
founder config setup                  # re-run interactive setup
founder models                        # fetch & list available models

Priority order

Environment variables > config file > interactive setup wizard
